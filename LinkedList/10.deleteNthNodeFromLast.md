## deleteKthNodeFromLast

-- "I'll use two pointers with a gap of k+1. First, I move the fast pointer k+1 steps ahead. Then I move both pointers together until fast reaches the end. At that point, slow will be right before the node to delete. I use a dummy node to handle the edge case where we need to delete the head."

#### why Add dummy ?
### Problem Without Dummy
When deleting the **head node** (Nth from end = length), we can't use the standard pattern.

**Example:** `1→2→3`, delete 3rd from end (delete head)


```swift
func deleteKthNodeFromLast(_ head: Node?, k: Int) -> Node? {
    let dummy = Node(data: 0)  // ✅ Add dummy
    dummy.next = head
    
    var p1: Node? = dummy       // ✅ Start from dummy
    var p2: Node? = dummy       // ✅ Start from dummy
    
    for _ in 0...k {            // ✅ Move k+1 steps
        p1 = p1?.next
    }
    
    while p1 != nil {           // ✅ Check p1, not p1?.next
        p1 = p1?.next
        p2 = p2?.next
    }
    
    p2?.next = p2?.next?.next   // ✅ Simplified
    
    return dummy.next           // ✅ Return dummy.next
}
```


To delete kth from end:
  Position to delete = (length - k)
  
Need to stop at: (length - k - 1) to delete next node

Gap between pointers = k + 1

Why k+1?
  fast at position (k+1)
  slow at position 0
  
When fast reaches end (position L):
  slow at position (L - k - 1) ✓
